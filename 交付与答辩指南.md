# 《呼吸岛 Liminal》交付与答辩指南

## 可交付内容

- `BreathIsland.xcodeproj`：visionOS / Apple Vision Pro 原型工程。
- `《呼吸岛》初赛作品说明文档(1).docx`：原始参赛说明。
- `呼吸岛-参考海报.jpg`：用户提供的视觉参考。
- `呼吸岛-海报-v2.png`：已放入 App 资源目录并用于主控侧栏的产品海报。
- `validation-artifacts/`：构建日志与 Vision Pro Simulator 运行截图。

## 在 Xcode 中运行

1. 打开 `BreathIsland.xcodeproj`。
2. Scheme 选择 `BreathIsland`，运行目标选择 `Apple Vision Pro`。
3. 按 Run，主窗口会显示在 Vision Pro Simulator 的共享空间中。
4. 在“治疗”页选择或按推荐启动场景，点击“进入沉浸空间”，即可打开 RealityKit 沉浸治疗空间。

## 3-5 分钟答辩演示顺序

1. 总览：说明 Vision Pro 是治疗主端，iPhone 只承接陪伴和信息共享；展示海浪呼吸的推荐理由、患者状态和主视觉。
2. 评估：切到“评估”，选择病种、疼痛性质、情绪和功能受限程度，生成个性化方案。
3. 推荐：说明本地规则引擎把临床情境映射到三种非药物辅助场景，且不替代医生诊断。
4. 治疗：切到“治疗”，展示冰火对抗、海浪呼吸、星屿冥想三个不同空间疗程；按推荐启动海浪呼吸并进入沉浸空间。
5. 反馈：推进疗程后进入“反馈”，讲解心率、HRV、呼吸频率与疼痛评分变化。
6. 医生端：切到“医生端”，展示风险建议、依从性、评估摘要和下一次启动建议，完成患者到临床协同的闭环。

## 验证记录

- Xcode 16.4 / visionOS 2.5 Simulator 的 clean build 和 incremental build 均通过。
- 已核验总览、治疗、反馈、医生端的真实 Simulator 截图；新版截图均保存在 `validation-artifacts/`。
- 构建时的 AppIntents metadata extraction 提示不影响本项目运行，因为工程未使用 App Intents。

## 原型边界

本原型的设备连接、生物反馈、家属消息和临床数据均为本地 Mock，旨在稳定呈现比赛所需的产品闭环。它是非药物疼痛辅助体验的设计验证，不构成诊断、治疗建议或医疗承诺。
